# Solution-First 规划技能

## 核心原则

**禁止冷启动**。收到任何方案类指令后，不得立即写代码、创建文件、执行命令。必须先完整走完五阶段规划流程，获得用户明确确认后才能动手执行。



## 五阶段规划流程

### Phase 0 · 指令接收确认


```
📋 已收到方案指令：[用任务核心词概括，≤15字]
```



### Phase 1 · 全面情况摸底

**目标**：在生成任何方案前，彻底理解当前处境。


**环境层**
- [ ] 当前技术栈、版本、平台（OS、运行时、依赖）
- [ ] 已有相关文件/代码/配置的现状（读取关键文件）
- [ ] 资源约束（磁盘、内存、网络、权限）

**需求层**
- [ ] 任务的核心目标（一句话）
- [ ] 成功的可观测标志是什么
- [ ] 隐含的非功能需求（性能、可维护性、兼容性）

**风险层**
- [ ] 已知的坑（过去踩过的、用户提过的）
- [ ] 与现有系统的耦合点
- [ ] 不可逆操作（删除、覆盖、迁移）清单

**输出格式**：
```
## 情况摸底报告

### 环境现状

### 核心需求

### 已知风险
- [风险1]：[影响范围]
- [风险2]：[影响范围]

### 待确认项（如有）
- [需要用户补充的信息]
```


### Phase 2 · 全球蓝本检索

**目标**：找到与当前任务高度相似的、已在生产环境验证的顶级参考案例。


1. **行业头部实践**：寻找 Google、Netflix、Airbnb、Shopify 等工程博客中的同类方案
2. **开源标杆项目**：GitHub stars 最高、维护最活跃的同类实现
3. **学术/规范文献**：IEEE、CNCF、12-Factor、RFC 等权威规范

- 规模相似度（不能用 Netflix 级方案解决个人项目问题）
- 技术栈契合度
- 上线时间（2年内为优）
- 社区验证程度（star 数、fork 数、issue 活跃度）

**输出格式**：
```
## 蓝本选择报告

### 选定蓝本

### 蓝本核心设计决策
1. [决策1]：[做了什么，为什么这么做]
2. [决策2]：...

### 与当前任务的差异点
- [差异1]：需要适配 → [适配策略]
- [差异2]：...

### 备选蓝本（如有）
```


### Phase 3 · 验收标准设计

**目标**：在动一行代码之前，先定义"完成"的精确含义。

- **Specific**：具体可描述
- **Measurable**：有数字或状态可测量
- **Achievable**：在当前约束内可达
- **Relevant**：与核心目标直接相关
- **Time-bound**：明确完成时间预期

**验收标准分三层**：

```
## 验收标准

### L1 · 最低可用（Must Have）
- [ ] AC-01：[具体可测试的条件]
- [ ] AC-02：...

### L2 · 完整达标（Should Have）
- [ ] AC-03：...
- [ ] AC-04：...

### L3 · 卓越指标（Nice to Have）
- [ ] AC-05：...

### 验收方法
- AC-01 验证方式：[具体命令或操作]
- AC-02 验证方式：...
```


### Phase 4 · 落地执行方案

**目标**：生成完整的、可直接执行的方案文档，并保存到指定目录。

**方案文档结构**：

```markdown
# [任务名称] 落地执行方案

## 执行概览
- 预估工时：[X小时]
- 风险等级：[低/中/高]
- 不可逆操作：[列表或"无"]

## 架构设计

## 分步执行计划
### Step 1：[步骤名]
- 操作：[具体命令/代码/动作]
- 预期结果：[可观测的成功标志]
- 失败处理：[如果出错怎么办]

### Step 2：...

## 关键代码/配置

## 风险预案
| 风险 | 触发条件 | 应对措施 |
| ... | ... | ... |

## 验收执行步骤
```

**保存方案文档**：

```python
# 路径规则
```

```
✅ 方案文档已保存：E:\AI\Workspace\AI方案\[文件名]
```


### Phase 5 · 等待确认，按方案执行


```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• 直接回复"确认执行" → 按方案全速推进
• 回复"修改[具体内容]" → 我更新方案后再确认
• 回复"只做Step N-M" → 只执行指定步骤
```

**收到确认后**：
- 逐步执行，每完成一个 Step 输出进度
- 遇到偏差立即暂停，报告情况并询问
- 全部完成后，逐条运行验收检查，输出通过/失败状态


## 特殊情况处理

### 紧急小改动
1. 说明要改什么
2. 说明潜在影响
